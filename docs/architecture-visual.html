01 — Overview

What is Vibe HQ?

Vibe HQ is the meta-project that sits above all other coding projects. It does not ship product — it governs the infrastructure that every product project inherits from. Think of it as the "platform team" for a one-person AI-augmented dev shop.

Every project that gets spun up comes pre-wired with agents, hooks, skills, MCPs, and a CLAUDE.md — not because they were configured manually, but because vibe-hq scaffolded them from templates. When the templates improve, future projects get the improvement. When a project discovers something worth keeping, vibe-hq harvests it and promotes it into the library.

Three templates cover the main project archetypes. Each template is a manifest.json plus CLAUDE.md and AGENTS.md templates. The manifest declares component references into the library — not copies of files.

Promotion is always human-gated. The harvest reports what it finds. A human session in vibe-hq decides whether a pattern is worth generalising. This keeps the library high-signal and prevents automated noise accumulation.

05 — Automation

Scheduled Jobs

Eight launchd jobs run autonomously. All are managed via Lingon Pro 10. Every script sources tools/scripts/env.sh to reconstruct the PATH that launchd strips. Jobs are silent on success — Telegram fires only when attention is needed.

Notification pattern: Success = silent. Attention needed = Telegram notification. Hard failure = Telegram + GitHub issue (deduplicated by title via create-issue.sh).

07 — OpenClaw

The Autonomous Agent

OpenClaw is a separate, autonomous agent running 24/7 on the claw macOS account. It is configured and governed from vibe-hq but operates independently. Think of it as a persistent executive assistant that lives alongside the Mac.

📧

Email Digests

Reads Thunderbird via thunderbird-mcp. Produces morning and evening email summaries via Telegram.

🌐

Web Research

Uses jina-reader and playwright for headless browsing, news monitoring, and on-demand research.

💬

Telegram Interface

YOUR_USER can send tasks, questions, and commands via Telegram. OpenClaw responds and acts.

💻

Coding Delegation

For code tasks, OpenClaw SSHs to YOUR_USER@localhost and spawns Claude Code. Never writes code itself.

Architecture

# OpenClaw v2026 stack
Runtime:    Node.js 22 + OpenClaw core
Gateway:    LiteLLM (model routing)
Models:     Anthropic Claude (primary)
             Alibaba Qwen (reference refresh, bulk ops)
             Gemini (fallback / research)

             jina-reader        → web fetch, search
             playwright         → headless browser

Security:
  binding:      ws://127.0.0.1:29741  (loopback only)
  auth:         token-based
  exec:         security = "deny"
  fs:           workspaceOnly = true
  firewall:     macOS blocks inbound

Config location:  /Users/YOUR_AGENT_USER/.openclaw/openclaw.json

SOUL.md governs everything. The OpenClaw identity file defines hard rules (never auto-approve purchases, never delete email, always CC YOUR_USER on client emails), communication style, and the boundary between what OpenClaw does autonomously vs what it escalates. Vibe HQ owns this file — changes are deployed via scripts/deploy-openclaw.sh.

10 — Market Intelligence

Daily Market Scanner

Three scheduled sessions scan global markets for short-term trending opportunities. Each session covers its regional window, surfaces setups with minimal downside risk, and writes a structured report to zz_handoffs/. Reports stop at buy signal — the setup is identified and the confirmation condition is stated, but no explicit buy/sell recommendations are made.

Signal discipline: Every opportunity entry includes a Buy setup if: field — the specific condition that would need to hold for entry (e.g. "holds above premarket high at open with volume confirmation"). The scanner identifies; you decide.

1 What the scanner looks for

Momentum setups

Stocks gapping on volume with a clear catalyst. Sector rotation signals. Index inclusion plays where the outcome is already confirmed.

News catalysts

Earnings surprises (beats or misses trading premarket), M&A announcements, analyst upgrades/downgrades, regulatory approvals, macro data surprises.

Low-risk setups

Situations where the outcome is largely known and the trade is riding confirmation. Arbitrage spreads. Confirmed catalysts where price hasn't fully moved.
